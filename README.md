## サービス概要
スケジュール忘れさせないために通知で怒られるアプリを作成検討。<br>
たくさんの機能を作るというより、スケジュールを簡単に作成できるアプリに通知で忘れさせないように工夫したい。

## このサービスへの思い・作りたい理由
私は仕事ではスケジュール管理はできるが、プライベートとなると、少しおろそかになる。<br>
なぜプライベートでは管理できずに仕事ではできるのかというと、もちろん、仕事だからというのもあるが、人間の感情的な部分があると思う。<br>
例えば、「怒られるのが嫌、迷惑をかけてしまうのが嫌」など。<br>
そのような人間の感情的な部分をうまく利用して、プライベートのスケジュールを管理したい。<br>
プライベートでもスケジュール管理をして、自分の人生を向上させたい。

## ユーザー層について
ユーザー層：予定を忘れやすい人(特に私と同じようにプライベートの予定を忘れてしまう人)<br>
自分自身が困っているためにアプリを作成しようと考えたので、自分と同じ困っている人を対象。

## サービスの利用イメージ
- ユーザーはスケジュールを簡単に登録することができる。(通知設定はマストにする)<br>
  →予定と通知時間だけで簡単に登録することができるので、使いやすい。通知の設定をする必要がない。

- 通知内容はその予定とその予定に対する感情を強く表現した文
  →感情的な文をで心を揺すぶることにより、予定を忘れにくくする。

## ユーザーの獲得について
予定を忘れやすい人に、簡単かつ忘れにくいスケジュール管理を提供したい。

## サービスの差別化ポイント・推しポイント
#### Outlookカレンダー(私が勤めている会社で使用可能)
#### 良い点
- Microsoft 365と連携でき、たくさんの便利な機能が使える。
- そもそもビジネス、企業ではMicrosoft 365をが使用されていることが多く、特にteamsなどとの連携は便利。(打ち合わせが多い部などでは)

#### 悪い点
- アプリが見にくい。私の会社では、最初使用していたが見にくいのでやめた人が多い。
- Outlookカレンダーとしてのアプリはなく、Outlook(メールアプリ)と統合しているので、わかりにくい。

### Googleカレンダー
#### 良い点
- スマホ、PCなどで同期されておりいつでもどこでも確認することができる。
- Googleの他のアプリ(Google Meet、Googleタスク)と連携して、より便利に使用できる。

#### 悪い点
- Googleアカウントが必要なので、作成していない人には一手間。
- 機能が多く、一部の機能しか使用していない人には恩恵が少ない。

### アプリの差別化ポイント
- 機能はスケジュール登録と通知でできるだけシンプルに。
- 通知の内容は予定だけではなく、感情的な文章も→忘れにくくする。


## 機能候補
### MVPリリース時に作っていたいもの
- LINE認証機能
- スケジュール登録機能
- 通知機能
  通知内容に生成AIでのコメントを自動入力。

### 本リリースまでに作っていたいもの
- 通知に気づかなかったり、反応しなかったら、自分のLINE友達に自動でメッセージを送信。
  →迷惑をかけてしまうという感情から忘れにくくする。もしくは友達に予定を教えてもらうようにする。
  (そもそも規約的にできなさそう→LINEbotを作成すれば間接的にできそう。モラル的にどうなのか→よくない気がする)

## 使用する技術スタック
### CRUD以外の実装機能予定イメージ
- LINEのapiを利用して、登録した自分のアカウントに予定の通知をする。
- その際、Gemini APIで感情的なコメントを通知内容に入れる。

### サービスを実装するための技術スタック
- 使用するフレームワーク:Ruby on Rails
- データベース:PostgreSQL
- デプロイ先:Render
- 使用予定のライブラリ:Devise / Omniauth
- 使用予定のAPI:LINE Messaging API / Gemini API

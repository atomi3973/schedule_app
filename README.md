## サービス概要
自己管理ができるようになるために通知に予定と忘れないように意識させるためのコメントを自動入力するアプリを作成検討。<br>
予定を簡単に作成するかつ、自己管理ができるような意識変えができるようなアプリにしたい。

## このサービスへの思い・作りたい理由
私は仕事ではスケジュール管理はできるが、プライベートとなると、少しおろそかになる。<br>
プライベートで歯医者の予約をしても、「痛くないし、まあいいか」と忘れてしまったり、当日になってもキャンセルしてしまうことがある。<br>
結局、次に痛くなって後悔するのを繰り返してしまう。<br>
一方、仕事の予定は絶対に守るし、「まあいいか」で流すことはない。<br>
なぜプライベートでは管理できずに仕事ではできるのかというと、<br>
プライベートは自己責任かつ、そもそも自己管理するもしないも選択できることが要因の一つだと思う。<br>
仕事ではスケジュール管理できるのだが、プライベートでは「まあいいか」と流してしまう。<br>
しかし、プライベートで「まあいいか」という気持ちで行動すると、知らず知らずのうちに誰かに迷惑をかけてしまい、<br>
その結果、自分にも不利益として返ってくる。<br>
なので自分のプライベートでの意識をできるだけ変えることができれば、スケジュール管理はよりうまくいき、<br>
自分の人生も向上していくと思っている。

## ユーザー層について
ユーザー層：予定管理が苦手で自分に甘くなってしまう傾向がある若者(10,20代)<br>
自分自身が困っているためにアプリを作成しようと考えたので、自分と同じ困っている人を対象。<br>
予定管理、自己管理が苦手な方。

## サービスの利用イメージ
- ユーザーは予定をテンプレート化して、ボタンで簡単に登録することができる。(通知設定はマストにする)<br>
  →予定と通知時間だけで簡単に登録することができるので、使いやすい。

- 通知内容はその予定とその予定に対するコメントを入れる
  →コメント内容でスケジュールを管理する意識を持たせて、予定を忘れにくくする。
   どのようなコメントが入るかは設定次第。

## ユーザーの獲得について
予定を忘れやすい人、自己管理ができない人に簡単かつ忘れにくいスケジュール管理を提供し、また意識を変えさせたい。

## サービスの差別化ポイント・推しポイント
#### Outlookカレンダー(私が勤めている会社で使用可能)
#### 良い点
- Microsoft 365と連携でき、たくさんの便利な機能が使える。
- そもそもビジネス、企業ではMicrosoft 365をが使用されていることが多く、特にteamsなどとの連携は便利。(打ち合わせが多い部などでは)

#### 悪い点
- アプリが見にくい。私の会社では、最初使用していたが見にくいのでやめた人が多い。
- Outlookカレンダーとしてのアプリはなく、Outlook(メールアプリ)と統合しているので、わかりにくい。

### Googleカレンダー
#### 良い点
- スマホ、PCなどで同期されておりいつでもどこでも確認することができる。
- Googleの他のアプリ(Google Meet、Googleタスク)と連携して、より便利に使用できる。

#### 悪い点
- Googleアカウントが必要なので、作成していない人には一手間。
- 機能が多く、一部の機能しか使用していない人には恩恵が少ない。

### アプリの差別化ポイント
- 機能はスケジュール登録と通知でできるだけシンプルに。
- 通知の内容は予定だけではなく、コメントも入れる。


## 機能候補
### MVPリリース時に作っていたいもの
- LINE認証機能
- 予定登録機能
  ボタンでテンプレを登録するようにする(約束事、美容院、病院、ご飯、支払いなど)、通知時間も設定
- 通知設定
  通知キャラ設定：動物(ライオン、カラス、亀)など愛着のあるキャラを選べるようにする
  コメント設定：ポジティブ、後悔、罪悪感、応援など
- 通知機能
  通知内容に生成AIでのコメントを自動入力。<br>
  内容はコメント設定ごとに違うようにする。<br>
  例えば予定内容が病院でコメント設定がポジティブの場合、「自分の体をキレイにして、気分をあげよう！いいことが起こりそう！」など<br>
  

### 本リリースまでに作っていたいもの
#### 予定のテンプレ作成機能
- 予定のテンプレを自分で作成できるようにする

#### コミットメント機能
- 予定完了数の可視化
  できれば、LINEで通知に返信するだけで、予定のステータスを完了にしたい<br>
  どれだけ予定通りにこなせたか、数を確認できるようにしたい。<br>
  →予定、通知キャラ設定、コメント設定ごとに成功数を確認できるようにし、どれが一番成功しているか確認できるようにする。<br>
  簡単なデータ分析などして、もっと見やすくできればいいがそこまでの知識がないため、将来的に考えておき、時間があれば実装。

- 連続予定成功率可視化
  予定がどれだけ連続で成功しているかを数字で可視化していく。

- 予定登録する際のアラート機能
  予定を登録する際に、失敗しそうな予定にはアラート。例えば、歯医者の予定達成率が50%を切っていたら、<br>
  この予定は達成率が少ないので、気を付ける旨のアラートを入れる。

- キャラのコメント
  キャラクターが様々なコメントをしていく。例えば、最近病院の予定がない場合は、「体が回復してよかった。」など。
  できるだけ、アプリを使いたくなるようなコメント。

## 使用する技術スタック
### CRUD以外の実装機能予定イメージ
- LINEのapiを利用して、登録した自分のアカウントに予定の通知をする。
- その際、Gemini APIでコメントを通知内容に入れる。
- その他、可視化、コミットメント機能などはLINEのapiなどを技術調査する必要がある。

### サービスを実装するための技術スタック
- 使用するフレームワーク:Ruby on Rails
- データベース:PostgreSQL
- デプロイ先:Render
- 使用予定のライブラリ:Devise / Omniauth
- 使用予定のAPI:LINE Messaging API / Gemini API
